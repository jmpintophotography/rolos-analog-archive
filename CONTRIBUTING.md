# Contributing

Bug fixes, accessibility improvements, translations and focused workflow improvements are welcome.

1. Fork the repository and create a short-lived branch.
2. Keep personal Firebase values and real archive exports out of commits.
3. Use fictional data in examples and tests.
4. Run `npm test`.
5. Describe the behavior before and after the change in the pull request.

The application intentionally has no build framework. Prefer small changes that preserve the static HTML, CSS and JavaScript architecture unless a larger migration has a clear user benefit.

Do not submit scanned film-package artwork unless you own it or have permission to redistribute it.
